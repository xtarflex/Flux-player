# Contributing to Flux Player

Welcome! We are excited that you want to contribute to Flux Player. This document outlines the process for contributing to our open-source project.

## Code of Conduct
Please review and abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

## How Can I Contribute?

### Reporting Bugs
* Ensure the bug was not already reported by searching on GitHub under Issues.
* If you're unable to find an open issue addressing the problem, open a new one. Be sure to include a title and clear description, as much relevant information as possible, and a code sample or an executable test case demonstrating the expected behavior that is not occurring.

### Suggesting Enhancements
* Open a new issue with the label "enhancement".
* Describe the feature clearly. Explain why this feature would be useful.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes (`npm run check` and `cargo check`).
4. Format your code (e.g., using Prettier for JS/Svelte and `rustfmt` for Rust).
5. Update documentation if necessary.
6. Issue that pull request!

## Local Development
Please refer to the developer onboarding section in `flux-player/README.md` for instructions on setting up your local environment.

Thank you for contributing!
